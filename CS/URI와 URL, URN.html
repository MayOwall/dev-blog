🚀 목표 : URI와 URL, URN에 대해 알아보기

URI

URI(Uniform Resource Identifier, 통합 자원 식별자)는 인터넷에 있는 자원을 나타내는 유일한 주소를 의미합니다. URI의 정의 및 구조는 RFC 3986이라는 문서에서 정의되었습니다. 일반적으로 URI는 아래와 같은 구조를 가지고 있습니다.

scheme:[//[user[:password]@]host[:port]][/path][?query][#fragment]

🤔 URI와 URL의 차이점

URL은 URI의 종류 중 하나를 의미합니다.
URI는 크게 URL과 URN으로 종류가 나뉘며, 현재 웹에서는 일반적으로 URL이 널리 통용되어 사용되고 있습니다.

URN

URN(Uniform Resource Name, 통합 자원 이름)은 이름을 통해 자원을 식별하는 방식을 의미합니다. URN은 아래와 같이 urn과 이름공간으로 구성되어 있으며, 주어진 이름 공간 안의 이름으로 자원을 식별합니다. 현재 웹에서는 거의 사용되지 않고 있습니다.

예) 도서 ISBN

urn:oasis:names:specification:docbook:dtd:xml:4.1.2

URL

URL(Uniform Resource Locator)은 네트워크 상에서 자원이 어디 있는지를 알려주기 위한 주소를 의미합니다. 즉 웹 페이지의 주소에 해당합니다. URL을 통해 웹 페이지 자원의 위치 뿐만 아니라 실시간 교통 정보 등 컴퓨터 네트워크 상의 모든 자원을 나타낼 수 있으며, 현재 웹에서 보편화되어 사용되고 있습니다.

URL은 아래와 같은 구조를 지니고 있습니다.

scheme://[userinfo@]host[:port][/path][?query][#fragment]
예시 ) https://www.google.com/search?=hello&hi=ko

scheme

사용하는 프로토콜을 의미합니다. 가장 대표적인 프로토콜로는 http(80포트), https(443 포트)가 있습니다.프로토콜이 정확히 명시되면 뒤의 port는 생략 가능합니다.

프로토콜 : 어떤 방식으로 자원에 접근할 것인가 하는 약속 규칙

https: http에 보안을 추가한 프로토콜 (HTTP Secure)

userinfo

URL에 사용자정보를 포함해서 인증해야할 때 사용하는 정보입니다. 현재는 거의 사용하지 않고 있습니다.

host

호스트명을 의미합니다. 도메인명 혹은 IP 주소를 직접 사용할 수 있습니다.

PORT

접속 포트를 의미합니다. 일반적으로 프로토콜의 명시와 함께 생략되며, 자주 사용되는 포트 번호로는 80(http), 443(https)등이 있습니다.

path

리소스가 있는 경로를 의미합니다. 계층적 구조를 이루고 있습니다.

query

웹서버에 재공하는 파라미터를 의미하며, key=value 형태를 지니고 있습니다. ?로 쿼리를 시작하고, &로 쿼리 추가가 가능합니다.

예시 ) ?q=hello&hl=ko

fragment

http 내부 북마크 등에 사용되는 정보로, 서버로 전송되지는 않습니다. #으로 시작합니다.
